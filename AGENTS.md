# 프로젝트 작업 규칙

이 파일의 규칙은 이 프로젝트를 수정하는 모든 작업에 적용한다.

## 기준 문서

1. 앱 제작의 최우선 기준은 `docs/Hotel_CallCenter_Chat_MVP_Design.md`다.
2. 구현 내용이 원본 사양서와 충돌하면 임의로 구현하지 않는다.
3. 원본 사양서에서 두 가지 이상으로 해석될 수 있거나 보안·데이터·사용 흐름에 영향을 주는 내용이 불명확하면 구현 전에 사용자에게 구체적으로 질문한다.
4. 사용자가 확정한 내용은 원본 사양서와 관련 설계 문서에 반영한 뒤 구현한다.

## 변경 이력

1. 모든 작업에서 `CHANGELOG_KO.md`를 함께 갱신한다.
2. 각 항목에는 수정 시각(Asia/Tokyo, UTC+09:00), 수정한 파일, 수정 내용, 수정 이유, 확인 방법을 기록한다.
3. 코드 변경 없이 조사만 수행한 경우에도 중요한 설계 결정이나 발견 사항은 기록한다.

## 코드 주석

1. 새로 작성하거나 수정하는 모든 코드에는 한국어 주석을 자세히 작성한다.
2. 주석에는 코드가 무엇을 하는지만 아니라 해당 검증·상태 전환·예외 처리가 필요한 이유를 설명한다.
3. 공개 함수, 클래스, API 처리기, WebSocket 이벤트 처리기에는 입력값, 반환값, 실패 조건을 한국어로 설명한다.
4. 보안 검증, 세션 만료, 권한 확인, 중복 메시지 방지, 트랜잭션 경계에는 반드시 한국어 주석을 둔다.
5. 코드와 다른 오래된 주석은 남기지 않고 코드 변경과 함께 갱신한다.

## 구현 및 검증

1. `ROADMAP.md`의 순서와 완료 조건을 따른다.
2. 구현 전 `docs/08_System_Blueprint.md`와 `docs/09_Analysis_Roadmap.md`에서 관련 구간을 확인한다.
3. 기능 구현 후 관련 테스트를 추가하고 실행 결과를 변경 이력에 기록한다.
4. 프런트엔드 제한만 신뢰하지 않고 권한, 세션 상태, 만료 시각, 입력 길이, 요청 빈도, 중복 여부를 서버에서 검증한다.
