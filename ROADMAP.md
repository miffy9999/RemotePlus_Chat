# 호텔–콜센터 실시간 채팅 MVP 제작 로드맵

## 1. 목표

객실 QR 링크로 접속한 투숙객과 인증된 콜센터 직원이 WebSocket을 통해 최대 15분 동안 안전하게 실시간 텍스트 상담을 진행하는 웹 MVP를 구현한다.

## 2. 구현 원칙

- 투숙객 웹, 상담원 웹, 서버를 분리한다.
- 서버는 REST API, WebSocket, 세션 만료 처리를 함께 제공하되 모듈 경계를 유지한다.
- 접근 키 원문과 비밀번호 원문은 저장하지 않는다.
- 세션 권한, 상태, 만료 시각, 메시지 길이, 요청 빈도, 중복 여부는 서버에서 최종 검증한다.
- 모든 메시지는 `sessionId`, 고유 ID, `clientMessageId`, 생성 시각을 가진다.
- MVP에서는 텍스트 메시지와 `SENT`/`FAILED` 상태만 구현한다.
- 초기 관리자 MVP보다 복잡한 통계·설정 화면, 파일 전송, 번역, AI 응답, 음성·영상 통화는 MVP 범위에서 제외한다.

## 3. 권장 기술 결정 체크포인트

구현 시작 전에 아래 항목을 확정한다.

- 프런트엔드: React 기반 SPA 또는 Next.js
- 백엔드: WebSocket과 REST를 함께 지원하는 Node.js/NestJS, FastAPI 등의 프레임워크
- 실시간 프로토콜: 표준 WebSocket 또는 Socket.IO
- 데이터베이스: PostgreSQL 권장
- ORM/마이그레이션 도구
- 인증: 상담원용 단기 Access Token과 갱신 방식
- 로컬 실행 및 배포: Docker Compose 사용 여부
- 테스트: 단위, API 통합, WebSocket 통합, E2E 도구

## 5. 권장 구현 순서

1. 데이터 모델과 상태 전환
2. 접근 키 및 상담원 인증
3. 상담 REST API
4. WebSocket 메시지 저장·전달
5. 세션 자동 만료
6. 투숙객 화면
7. 상담원 화면
8. 재연결·알림·보안 강화
9. 통합/E2E 테스트와 운영 문서

## 6. 주요 위험과 선제 대응

| 위험 | 대응 |
|---|---|
| 클라이언트 타이머 조작 | 모든 만료 판단을 서버 시각과 `expiresAt` 기준으로 수행 |
| 메시지 중복 저장 | 세션별 `clientMessageId` 고유 제약과 멱등 처리 |
| 상담 간 메시지 유출 | 서버에서 사용자·세션 권한 검사 후 방 단위 전송 |
| 접근 키 노출 | 원문 미저장, 만료·비활성화 지원, 로그 마스킹 |
| 재연결 중 메시지 누락 | 마지막 수신 시점 이후 메시지 REST 재조회 |
| 서버 재시작 시 만료 누락 | DB 상태를 기준으로 만료 대상 재검색 |

## 7. MVP 이후 후보

QR 생성·미리보기·다운로드·삭제·재생성, 이미지/파일, 자동 번역, 만족도 조사, 상담 배정·인계, 고급 관리자 기능, 통계 대시보드, 모바일 앱, 외부 알림, AI 답변 추천은 MVP 승인 후 별도 에픽으로 관리한다.
