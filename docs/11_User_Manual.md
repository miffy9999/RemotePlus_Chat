# 호텔 콜센터 채팅 MVP 사용 설명서

## 1. 실행 준비

1. Docker Desktop을 실행한다.
2. 프로젝트 루트에서 `.env.example`을 `.env`로 복사하고 `JWT_SECRET`을 긴 임의 문자열로 변경한다.
3. 다음 명령을 순서대로 실행한다.

```powershell
docker compose up -d postgres
pnpm install
pnpm db:generate
pnpm db:migrate
pnpm db:seed
pnpm dev
```

서버 상태는 `http://127.0.0.1:4000/api/health`에서 확인한다. `status`와 `database`가 모두 `ok`이면 준비된 상태다.

## 2. 관리자 사용법

접속 주소: `http://127.0.0.1:5173/admin/login`

개발용 초기 계정은 `admin / Admin1234!`이다. 운영에서는 반드시 비밀번호와 JWT 비밀키를 변경한다.

### Agent 추가

1. Agent 관리에서 이름, 로그인 ID, 비밀번호를 입력한다.
2. 비밀번호는 영문과 숫자를 포함해 8자 이상 입력한다.
3. `Agent 추가`를 누르고 목록에 계정이 나타나는지 확인한다.

### 호텔과 룸 추가

1. 호텔 이름을 입력하고 `호텔 추가`를 누른다.
2. 룸을 추가할 호텔을 선택한다.
3. 객실 번호를 입력하고 `룸 추가`를 누른다.
4. 호텔 필터를 선택하면 해당 호텔의 룸만 표시된다.

`QR 관리`의 `준비 중` 버튼은 의도적으로 비활성 상태다. QR 생성·다운로드·폐기·재발급은 MVP 이후 기능이다.

## 3. Agent 사용법

접속 주소: `http://127.0.0.1:5173/login`

개발용 초기 계정은 `agent01 / Agent1234!`이다.

1. 로그인 후 대기 상담 목록을 확인한다.
2. `상담 수락`을 누르면 내 진행 상담으로 이동한다.
3. 객실·호텔·언어·연결 상태·남은 시간을 확인한다.
4. 아래 입력창에서 메시지를 작성하고 전송한다.
5. 종료할 때 `상담 종료`를 누르고 화면 확인창에서 한 번 더 종료한다.
6. 종료 상담의 `기록 보기`로 기존 메시지를 읽을 수 있다.

새 대기 상담이 증가하면 짧은 알림음이 난다. 브라우저가 소리를 차단한 경우 한 번 화면을 클릭한 뒤 사용한다.

## 4. 투숙객 사용법

MVP 테스트 주소: `http://127.0.0.1:5174/?accessKey=demo-room-access-1201`

1. 사용할 언어를 선택한다.
2. 개인정보 안내를 읽고 확인란을 선택한다.
3. `상담 시작`을 누른다.
4. 상담원이 수락할 때까지 대기한다.
5. 연결되면 메시지를 입력해 전송한다.
6. 남은 시간이 0이 되거나 Agent가 종료하면 입력창이 자동으로 잠긴다.

새로고침해도 같은 탭에서는 진행 중 상담과 메시지가 복구된다. 다른 탭은 별도 상담 탭으로 취급한다.

## 5. 상태와 오류 이해

- `WAITING`: 상담원 연결 대기
- `ACTIVE`: 상담 진행 중
- `CLOSED`: 상담원이 종료
- `EXPIRED`: 생성 후 15분이 지나 서버가 자동 종료
- 연결 끊김/재연결 중: 네트워크 상태를 확인하고 잠시 기다린다.
- 접근 오류: 링크의 접근 키가 잘못됐거나 비활성화됐다.
- 요청이 너무 많음: 1분 정도 기다린 뒤 다시 시도한다.

비밀번호, 카드번호, 결제 비밀번호 같은 민감정보는 채팅에 입력하지 않는다.

## 6. 운영자 점검과 테스트

```powershell
pnpm lint
pnpm test
pnpm --filter @hotel-chat/server test:phase2
pnpm --filter @hotel-chat/server test:phase45
pnpm build
```

문제가 생기면 먼저 `/api/health`, 서버 터미널의 JSON 로그, PostgreSQL 컨테이너 상태를 확인한다. 자동 만료 작업은 서버 시작 직후 한 번 실행되고 이후 5초마다 DB의 만료 대상을 찾는다.
