# 호텔 콜센터 채팅 MVP 사용 매뉴얼

## 전체 상담 로그

- Agent와 관리자는 각 화면 최하단의 `전체 상담 로그`에서 모든 상담원의 완료 기록을 확인할 수 있습니다.
- `호텔 필터`로 특정 호텔만 선택할 수 있으며 기록이 많으면 목록 블록 안에서 아래로 스크롤합니다.
- `기록 보기`를 누르면 실시간 상담방과 분리된 읽기 전용 창에 저장된 메시지가 표시됩니다. 다른 Agent가 현재 진행 중인 상담은 공동 로그에 표시되지 않습니다.
- 대기·진행 목록은 5초마다 갱신하지만 완료 로그는 최초 진입·상담 상태 변경·60초 주기에만 갱신하므로 무료 서버에 전체 기록을 반복 요청하지 않습니다.

## 1. Docker로 처음부터 실행

가장 간단한 방법은 프로젝트 루트의 `서버_실행.cmd`를 더블클릭하는 것이다. 배치 파일이 Docker Desktop 실행 여부와 `.env` 파일을 확인하고 전체 컨테이너를 빌드·실행한 뒤 접속 주소를 표시한다.

1. Docker Desktop을 실행한다.
2. PowerShell에서 프로젝트 폴더로 이동한다.
3. 환경변수 파일을 만든 뒤 전체 컨테이너를 빌드하고 실행한다.

```powershell
cd C:\Users\전성현\Desktop\00_2026_Summer_Internship\Remote_plus\chat_app
Copy-Item .env.example .env
docker compose up -d --build
docker compose ps
```

Docker는 현재 프로젝트 폴더의 소스를 이미지에 복사하여 실행한다. PC의 다른 위치에 있는 예전 프로젝트를 참조하지 않는다. 최초 실행 시 `migrate` 컨테이너가 DB 구조를 갱신하고 개발용 초기 데이터를 넣은 다음 API 서버가 시작된다.

상태 확인 주소는 `http://127.0.0.1:4100/api/health`이다. Docker 내부 서버는 4000번을 사용하지만 Windows 호스트에서는 예약 포트 충돌을 피하기 위해 기본 4100번으로 연결한다. 응답의 `status`와 `database`가 `ok`이면 정상이다.

## 2. 관리자 사용법

접속 주소: `http://127.0.0.1:5173`

공통 직원 로그인 화면에 관리자 계정을 입력하면 서버가 `ADMIN` 역할을 확인하고 관리자 페이지로 자동 이동한다. 직접 로그인 주소는 `http://127.0.0.1:5173/login`이다.

로그인 ID 입력란은 기본적으로 빈 상태로 열린다. `아이디 저장`을 선택하고 로그인하면 다음 방문에 ID만 복원되고, `로그인 정보 저장`을 선택하면 ID 복원과 함께 브라우저 비밀번호 관리자에 저장을 요청한다. 두 버튼을 다시 누르면 선택을 해제할 수 있으며 비밀번호는 앱의 localStorage에 저장하지 않는다.

무료 테스트 관리자 계정: `admin / admin`

로그인 정보는 현재 브라우저 탭의 `sessionStorage`에 저장되므로 새로고침 후에도 유지된다. 명시적으로 로그아웃하거나 탭을 닫으면 제거된다.

### Agent 추가

1. 이름, 로그인 ID, 비밀번호를 입력한다.
2. 무료 테스트에서는 비밀번호 길이와 문자 조합을 제한하지 않는다.
3. `Agent 추가`를 누르고 목록에 계정이 표시되는지 확인한다.

### Agent 삭제

1. Agent 목록의 `삭제` 버튼을 누른다.
2. 이름과 로그인 ID를 확인하고 확인창에서 승인한다.
3. 계정은 삭제되지만 기존 상담 기록은 보존되며 담당 Agent 연결만 제거된다.

### 호텔과 룸 추가 및 고객 주소 확인

1. 호텔 이름을 입력하고 `호텔 추가`를 누른다.
2. 룸을 추가할 호텔을 선택하고 객실 번호를 입력한다.
3. `룸 추가`를 누르면 접근키와 고객 주소가 자동 생성된다.
4. 룸 테이블의 `상담 링크 열기`를 누르면 고객 화면이 새 탭에서 열린다.
5. `주소 복사`를 누르면 고객에게 전달할 전체 주소가 클립보드에 복사된다.
6. 호텔 필터를 선택하면 해당 호텔의 룸만 볼 수 있다.

### 룸과 호텔 삭제

- 룸 행의 `삭제` 버튼은 해당 룸의 접근키, 상담, 메시지를 함께 영구 삭제한다.
- `선택 호텔 삭제`는 현재 룸 추가 선택창에 선택된 호텔과 모든 하위 룸·접근키·상담·메시지를 영구 삭제한다.
- 삭제 전 확인창에 표시되는 호텔·룸 이름과 영향 범위를 반드시 확인한다. 삭제된 데이터는 복구할 수 없다.

각 룸의 `QR 보기` 버튼을 누르면 해당 고객 주소를 담은 고정 QR을 미리 볼 수 있다. 호텔명과 객실 번호를 확인한 뒤 `PNG 다운로드`를 눌러 약 1024px 고해상도 이미지를 저장한다. QR에는 여백이 포함되어 있으므로 잘라내지 말고 인쇄한 뒤 실제 휴대전화로 스캔 시험을 한다. 가급적 가로·세로 3cm 이상으로 인쇄한다.

이 QR은 정기 갱신되지 않는다. 객실 또는 접근 키를 삭제하거나 고객 웹 도메인을 바꾸면 기존 인쇄물로 접속할 수 없으므로, 상업 인쇄 전에 장기간 사용할 운영 도메인을 반드시 확정한다.

## 3. Agent 사용법

접속 주소: `http://127.0.0.1:5173`

관리자와 같은 공통 직원 로그인 화면에 Agent 계정을 입력하면 서버가 `AGENT` 역할을 확인하고 상담 목록으로 자동 이동한다. 직접 로그인 주소는 `http://127.0.0.1:5173/login`이다.

무료 테스트 Agent 계정: `agent01 / agent01`

무료 테스트 관리자 계정: `admin / admin`

두 비밀번호는 공개 테스트 편의를 위한 약한 값이다. 외부 공개 또는 상업 운영을 시작하기 전에 강한 개별 비밀번호로 반드시 교체한다.

로그인 정보는 판별된 역할별로 현재 탭의 `sessionStorage`에 저장된다. 24시간 콜센터 운영 중에는 토큰 시간 만료로 재로그인을 요구하지 않지만, 계정이 삭제·비활성화되면 다음 API 요청이나 WebSocket 연결에서 즉시 공통 로그인 화면으로 돌아간다.

Render 무료 서버가 15분 이상 요청을 받지 않으면 휴면 상태가 되어 첫 로그인에 약 1분이 걸릴 수 있다. 로그인 페이지를 열면 서버 기동 요청을 먼저 보내며 4초 이상 걸릴 때 화면에 기동 중 안내가 표시된다. 이는 오류가 아니며 무료 플랜의 휴면 제한 자체는 앱에서 제거할 수 없다.

관리자와 Agent는 화면 상단의 `비밀번호 변경`을 눌러 현재 비밀번호, 새 비밀번호와 확인값을 입력할 수 있다. 성공하면 모든 PC에 남아 있던 해당 계정 토큰이 무효화되고 현재 화면도 로그아웃된다. 새 비밀번호로 다시 로그인한다. 기본 테스트 비밀번호를 그대로 유지하면 Chrome의 유출 비밀번호 경고는 계속 표시될 수 있다.

각 비밀번호 입력칸 오른쪽 눈 아이콘을 누르면 해당 칸의 입력값만 표시하거나 다시 숨길 수 있다.

1. 로그인하여 대기 상담 목록을 확인한다.
2. 화면 상단에서 표시 언어를 바꾸거나 현재 Agent 이름을 확인하고 로그아웃할 수 있다. 현재 상담 목록으로 다시 이동하기만 하던 왼쪽 메뉴는 표시하지 않는다.
3. `상담 수락`을 누르면 진행 상담으로 이동한다.
4. 메시지를 보내고 고객과 실시간으로 대화한다.
5. 종료 시 `상담 종료`를 누르고 확인창에서 한 번 더 확인한다.
6. 종료 상담은 `기록 보기`에서 다시 확인할 수 있다.

새 메시지는 채팅창의 마지막 메시지로 자동 스크롤된다. 상담을 종료하거나 목록으로 돌아갈 때는 전체 페이지가 맨 위로 이동하지 않고 기존 목록 위치를 복원한다. 모바일에서는 상담 목록이 카드 형태로 표시되고 채팅 화면이 휴대폰 화면 전체 높이를 사용한다.

종료·만료된 상담 기록과 메시지는 종료 시각부터 30일 동안만 보관한 뒤 자동 삭제된다. 진행 중이거나 대기 중인 상담은 이 정리 작업에서 삭제하지 않으며, 삭제된 기록은 화면이나 DB에서 복구할 수 없다.

새 상담이 접수되거나 이 계정이 담당 중인 어느 상담에서든 고객 메시지가 오면, 현재 열어 둔 상담과 관계없이 화면 우측 상단에 호텔·객실과 메시지 미리보기가 포함된 팝업이 약 8초간 표시된다. 새 상담 팝업의 `대기 상담 보기`를 누르면 상담을 자동 수락하지 않고 목록으로 이동한다.

알림음은 기본으로 꺼져 있다. 대기 상담 영역의 `알림음 켜기`를 누르면 짧은 시험음이 재생되고 이후 새 상담·고객 메시지에 소리가 난다. 다시 `알림음 끄기`를 누르면 팝업은 유지하면서 소리만 끈다. 선택은 같은 PC와 브라우저에 저장된다.

Agent 탭이나 브라우저 창을 보고 있지 않을 때 새 상담 또는 고객 메시지가 오면 탭 제목이 새 채팅 도착 문구로 깜빡인다. Agent 탭을 다시 열거나 창에 초점을 맞추면 원래 `RemotePlus Agent` 제목으로 돌아온다. 진행 상담 목록에서는 마지막 메시지가 가장 최근인 상담이 맨 위에 표시된다.

탭을 다른 창 뒤에 두고도 알림을 받으려면 Agent 목록의 `브라우저 알림 켜기`를 누르고 브라우저 권한을 허용한다. Edge에서는 HTTPS 운영 주소로 접속해야 하며 서비스 워커 등록이 완료되어야 한다. 차단했다면 주소창 왼쪽 사이트 정보에서 알림을 허용하고 새로고침한다. 권한을 차단해도 앱 화면 안 팝업, 제목 깜빡임과 선택한 알림음 설정은 계속 동작한다.

## 4. 고객 사용법

고객은 관리자가 룸 테이블에서 복사한 주소로 접속한다.

1. 언어를 선택한다.
2. 개인정보 안내를 읽고 동의한다.
3. `상담 시작`을 누르고 Agent의 수락을 기다린다.
4. 연결 후 메시지를 보낸다.
5. Agent 수락 뒤 15분이 지나거나 고객·Agent가 종료하면 입력창이 잠긴다. 수락 전 대기 시간은 15분에 포함되지 않는다.
6. 고객이 직접 끝내려면 상단 `상담 종료`를 누르고 확인창에서 확정한다. 단순히 탭을 닫거나 연결이 잠시 끊긴 것은 상담 종료로 처리되지 않는다.

상담 요청 후 탭을 닫아도 같은 휴대전화와 브라우저에서 객실 QR을 다시 읽으면 진행 중인 `WAITING` 또는 `ACTIVE` 상담으로 자동 복귀한다. 상담이 종료·만료됐거나 서버가 저장 토큰을 거부하면 기존 복구 정보는 자동 삭제되고 새 상담 시작 화면이 표시된다.

## 5. 로그, 종료, 완전 초기화

```powershell
# 실시간 로그 보기(Ctrl+C는 로그 보기만 종료)
docker compose logs -f

# 컨테이너 종료(DB 데이터 유지)
docker compose down

# 컨테이너와 DB 데이터를 모두 삭제
docker compose down -v --remove-orphans
```

완전 초기화 후 `docker compose up -d --build`를 실행하면 빈 DB에서 마이그레이션과 시드가 다시 수행된다. `down -v`로 삭제한 상담 데이터는 복구되지 않는다.

## 6. 환경변수와 보안

- `JWT_SECRET`: 로그인 및 고객 토큰 서명 비밀값
- `ACCESS_KEY_ENCRYPTION_SECRET`: 관리자 페이지에서 룸 주소를 복원할 때 사용하는 암호화 비밀값
- `GUEST_PUBLIC_URL`: 고객에게 보여줄 고객 웹의 기준 주소(로컬 기본값 `http://localhost:5174`)

직원·관리자 인증은 탭 단위 `sessionStorage`를 사용한다. 직원 로그인 저장 모드와 ID, 고객 상담 복구 정보와 UI 언어 환경설정은 같은 기기 재접속을 위해 `localStorage`를 사용하며 직원 비밀번호는 브라우저 비밀번호 관리자에 위임한다. 상담 종료·만료·인증 거부 시 고객 복구 정보는 자동 삭제된다. 현재 광고·분석용 쿠키는 사용하지 않는다.

운영 환경에서는 `.env.example`의 기본값을 반드시 긴 무작위 값으로 변경한다. 특히 `ACCESS_KEY_ENCRYPTION_SECRET`이 바뀌면 기존 룸 주소를 복호화할 수 없으므로 안전하게 백업한다. `.env` 파일은 Git에 올리지 않는다.
