## 객실 고정 QR 기능

- [x] 기존 룸 고객 URL을 이용한 고정 QR 생성
- [x] 호텔명·객실 번호·고객 주소가 포함된 QR 미리보기
- [x] 오류 정정 H·4모듈 여백을 적용한 약 1024px 고해상도 PNG 다운로드
- [x] 모바일 관리자 화면에서도 사용할 수 있는 QR 모달
- [x] 상업 인쇄 전 운영 도메인 확정·실기기 스캔·3cm 이상 인쇄 안내
- [x] MIT `qrcode` 라이선스와 전체 production 의존성 분포 고지

범위 제외: 고정 인쇄 운영 결정에 따라 QR은 정기 만료·갱신하지 않으며 폐기·재발급 UI도 제공하지 않는다. 객실이나 접근 키 삭제 또는 고객 도메인 변경 시 기존 인쇄물은 사용할 수 없다.

## 갱신 규칙

1. 기능 작업을 시작할 때 관련 항목의 현재 상태를 확인한다.
2. 구현과 검증이 끝난 뒤 체크 상태와 설명을 실제 결과에 맞게 변경한다.
3. 새 기능이면 알맞은 구역에 항목을 추가한다.
4. `CHANGELOG_KO.md`에 변경 시각, 수정 파일, 이유, 검증 결과를 함께 기록한다.
5. 기능의 사용자 흐름이나 구조가 바뀌면 README, 사용 매뉴얼, 시스템 설계도도 함께 갱신한다.
