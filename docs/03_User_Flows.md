# 사용자 흐름

## 투숙객 흐름

1. 테스트용 링크의 `accessKey`로 객실 접근 권한을 확인한다.
2. 일본어를 기본으로 일본어·영어·한국어·중국어 중 상담 언어를 선택하고 저장 안내와 민감정보 입력 금지에 동의한다.
3. 상담 시작을 누르면 한 객실에 하나의 열린 상담만 생성된다.
4. `WAITING` 동안 상담원 연결을 기다리고, `ACTIVE`부터 메시지를 보낸다.
5. 연결이 끊겼다가 복구되면 REST 이력을 다시 불러와 누락 메시지를 복원한다.
6. 상담원이 종료하면 입력을 잠그고 종료 안내와 기존 대화를 유지한다.

상담 생성 후 세션 ID·게스트 토큰·만료 시각을 접근 키별 `localStorage`에 저장한다. 같은 기기에서 QR을 다시 읽으면 서버가 토큰을 재검증하고 `WAITING` 또는 `ACTIVE` 상담으로 복귀한다. 손상·만료·종료·인증 거부된 복구 정보는 해당 접근 키 항목만 제거하고 언어 선택·동의 화면으로 안전하게 복구한다.

## 상담원 흐름

1. 공통 직원 로그인 화면에 Agent 계정으로 로그인하면 역할 확인 후 Agent 상담 목록으로 자동 이동한다. ADMIN 계정은 같은 화면에서 관리자 페이지로 이동한다.
2. 대기 목록을 5초 간격으로 갱신하고 새 상담이 생기면 호텔·객실이 표시된 화면 팝업을 제공한다. 알림음은 기본으로 꺼져 있으며 Agent가 직접 켜거나 끌 수 있다. Agent가 권한을 허용하고 탭이 백그라운드라면 브라우저 시스템 알림도 표시하며, 비활성 탭 제목은 새 채팅 도착 문구와 원래 제목을 번갈아 표시한다.
3. 상담 수락 후 객실·호텔·언어·연결 상태·남은 시간을 확인한다.
4. 투숙객과 실시간 메시지를 교환한다. 이 Agent에게 배정된 모든 진행 상담에서 고객 메시지가 도착하면 현재 열어 둔 상담과 관계없이 내용 미리보기가 포함된 화면 팝업을 표시하고 해당 상담을 최근 채팅 목록 맨 위로 올린다.
5. 화면 내 확인창에서 상담을 종료한다.
6. 종료 목록의 기록 보기로 저장된 대화를 읽는다.

관리자는 룸 목록의 `QR 보기`에서 고객 URL을 담은 고정 QR을 확인하고 PNG로 다운로드한다. 투숙객은 객실에 비치된 QR을 스캔하며, 이후 흐름은 `?accessKey=...` 고객 URL과 동일한 서버 접근 검증 경로를 사용한다. QR은 정기 갱신하지 않으며 객실 또는 접근 키가 삭제되면 사용할 수 없다.
