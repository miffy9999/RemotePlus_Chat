# 사용자 흐름

## 공동 상담 로그 확인

1. Agent 또는 관리자가 직원 계정으로 로그인한다.
2. 화면 하단의 전체 상담 로그에서 모든 Agent의 종료·만료 상담을 확인한다.
3. 필요하면 호텔 필터를 선택하고, 누적 기록은 블록 내부에서 스크롤한다.
4. 기록 보기를 선택해 저장된 메시지를 읽는다. 타 Agent의 진행 중 상담은 공동 로그에 포함하지 않는다.

## 투숙객 흐름

1. 테스트용 링크의 `accessKey`로 객실 접근 권한을 확인한다.
2. 일본어를 기본으로 일본어·영어·한국어·중국어 중 상담 언어를 선택하고 저장 안내와 민감정보 입력 금지에 동의한다.
3. 상담 시작을 누르면 호텔 안내문을 첫 SYSTEM 메시지로 포함한 한 객실 하나의 열린 상담이 생성된다.
4. `WAITING`에서도 문의를 즉시 보내 DB에 저장한다. Agent가 대화를 열면 담당자만 배정되고, 담당 Agent의 첫 답변 메시지가 DB에 저장된 시점부터 15분의 제한 시간이 시작된다.
5. 연결이 끊겼다가 복구되면 REST 이력을 다시 불러와 누락 메시지를 복원한다.
6. 상담원이 종료하면 입력을 잠그고 종료 안내와 기존 대화를 유지한다.

상담 생성 후 세션 ID·게스트 토큰·만료 시각을 접근 키별 `localStorage`에 저장한다. 같은 기기에서 QR을 다시 읽으면 서버가 토큰을 재검증하고 `WAITING` 또는 `ACTIVE` 상담으로 복귀한다. 손상·만료·종료·인증 거부된 복구 정보는 해당 접근 키 항목만 제거하고 언어 선택·동의 화면으로 안전하게 복구한다.

## 상담원 흐름

1. 공통 직원 로그인 화면에 Agent 계정으로 로그인하면 `/agent`, ADMIN 계정으로 로그인하면 Dashboard가 아닌 ADMIN 전용 Agent 조회 화면 `/admin/agent`로 이동한다.
2. 비밀번호를 바꿀 때는 상단의 `비밀번호 변경`에서 현재 비밀번호와 새 비밀번호를 입력한다. 성공하면 해당 계정의 기존 로그인 토큰을 모두 무효화하고 공통 로그인 화면으로 돌아간다.
3. 대기 목록을 5초 간격으로 갱신한다. Agent 화면이 보이고 포커스된 동안에는 팝업을 만들지 않으며, 탭이 숨겨졌거나 다른 창을 사용하는 경우에만 호텔·객실이 표시된 무음 팝업을 제공한다. Agent가 권한을 허용하고 탭이 백그라운드라면 무음 브라우저 시스템 알림도 표시하며, 비활성 탭 제목은 새 채팅 도착 문구와 원래 제목을 번갈아 표시한다.
4. LINE형 `Current chat room`에서 새 문의를 열면 별도 수락 화면 없이 원자 배정되며 객실·호텔·언어·연결 상태·남은 시간을 확인한다.
5. 투숙객과 실시간 메시지를 교환한다. 이 Agent에게 배정된 모든 진행 상담에서 고객 메시지가 도착하면 해당 상담을 최근 채팅 목록 맨 위로 올리고, Agent 화면이 비활성 상태일 때만 내용 미리보기 팝업을 표시한다. ADMIN이 선택한 진행 상담은 읽기 전용 WebSocket 구독으로 Agent와 Guest의 새 메시지를 즉시 확인한다.
6. 화면 내 확인창에서 상담을 종료한다.
7. 종료 목록의 기록 보기로 저장된 대화를 읽는다.

## 관리자 Dashboard와 Agent 조회 화면 이동

1. ADMIN 계정으로 로그인하거나 저장 인증을 복구하면 `/admin/agent`가 열린다.
2. `/admin/agent`에서는 모든 호텔의 대기·진행 상담과 전체 종료 로그를 읽기 전용으로 조회하며 목록·상세에 실제 담당 Agent를 표시한다.
3. ADMIN Agent 화면의 `관리자 페이지` 버튼으로 `/admin` Dashboard에 직접 이동한다.
4. Dashboard에는 `호텔 관리 / Agent 관리`만 두고, 로고 오른쪽 `Agent 페이지` 버튼으로 `/admin/agent`에 돌아간다.
5. 두 버튼은 고정 route를 사용하며 ADMIN 인증과 전체 상담 조회 권한을 유지한다. 일반 Agent는 관리자 두 경로와 버튼에 접근할 수 없다.

관리자는 룸 목록의 `QR 보기`에서 고객 URL을 담은 고정 QR을 확인하고 PNG로 다운로드한다. 투숙객은 객실에 비치된 QR을 스캔하며, 이후 흐름은 `?accessKey=...` 고객 URL과 동일한 서버 접근 검증 경로를 사용한다. QR은 정기 갱신하지 않으며 객실 또는 접근 키가 삭제되면 사용할 수 없다.
