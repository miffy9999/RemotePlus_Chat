# 초기 화면 구성

## 직원 통합 진입 구조

초기 운영 화면은 역할 선택 없이 관리자·Agent 공통 로그인 폼을 표시한다. 로그인 성공 응답의 역할이 `ADMIN`이면 ADMIN 전용 Agent 조회(`/admin/agent`), `AGENT`이면 일반 상담(`/agent`)으로 자동 이동한다. Dashboard(`/admin`)는 ADMIN Agent 화면의 버튼으로만 들어가며 URL과 권한 검사는 역할별로 분리한다.

로그인 ID 입력란은 특정 테스트 계정을 기본값으로 채우지 않는다. 사용자가 `아이디 저장` 또는 `로그인 정보 저장`을 선택한 경우에만 이전 ID를 표시한다. `아이디 저장`은 ID만 앱 로컬 설정에 기록하고, `로그인 정보 저장`은 비밀번호를 앱 저장소에 남기지 않은 채 브라우저의 표준 비밀번호 관리자에 저장을 요청한다. 선택 중인 버튼을 다시 누르면 저장 모드를 해제한다.

관리자와 Agent 상단에는 `비밀번호 변경`을 제공한다. 모달에서 현재 비밀번호, 새 비밀번호, 새 비밀번호 확인을 입력하며 브라우저가 비밀번호 변경 폼으로 인식하도록 `current-password`와 `new-password` 자동완성 의미를 사용한다. 성공하면 기존 직원 토큰을 모두 폐기하고 공통 로그인으로 이동한다.

## 관리자 페이지

Dashboard 헤더 왼쪽에는 `REMOTE+ Dashboard` 로고와 `Agent 페이지` 버튼을 이 순서로 붙여 표시한다. `Agent 페이지`는 ADMIN 전용 `/admin/agent`로 직접 이동하며, 일반 Agent에는 Dashboard와 이 버튼을 모두 노출하지 않는다.

상단에는 등록 Agent·호텔·룸·Guest 주소 수를 요약 지표로 표시한다. 그 아래는 왼쪽 관리 메뉴와 오른쪽 작업 패널로 나누고 `호텔 관리`, `Agent 관리` 중 하나만 표시한다. Dashboard 전용 상담 Log는 제공하지 않으며 상담 기록은 ADMIN Agent 화면의 `Log` 탭에서 확인한다. 데스크톱 관리 메뉴는 접기·펴기를 지원하고 모바일에서는 가로 탭으로 전환한다.

### 호텔 관리

- 호텔·룸 관리와 호텔별 자동 안내문을 하나의 호텔 관리 카드 안에서 구분선으로 나눈 연속 하위 섹션으로 배치
- 강조형 호텔 추가와 등록 호텔 빠른 선택
- 등록 호텔 칩과 룸 추가용 호텔 선택을 카드 전체의 공통 선택값으로 사용하고 자동 안내문에는 중복 호텔 선택창을 두지 않음
- 상단 공통 호텔을 바꾸면 자동 안내문 편집값도 같은 호텔·현재 언어 원문으로 즉시 전환
- 첫 번째 `전체 호텔` 버튼과 뒤따르는 호텔별 선택 버튼
- 선택한 호텔에 룸 추가
- 룸 목록 테이블 조회
- 하위 안내문 카드의 `日本語 / English` 탭으로 호텔별 Guest 첫 자동 안내문 편집

권장 룸 테이블 열은 `호텔`, `룸 번호`, `상태`, `생성일`, `QR 관리`다.

### Agent 관리

- Agent 목록 조회
- Agent 추가
- 기본 계정 상태 표시

## QR 관리

룸 테이블 마지막 열의 QR 버튼에서 다음 기능을 제공한다.

- 룸별 고정 QR 코드 미리보기
- 약 1024px 고해상도 PNG 이미지 다운로드
- 호텔명·객실 번호·현재 고객 URL 확인
- 인쇄 전 운영 도메인 확정 및 휴대전화 스캔 시험 안내

QR은 현재 객실 고객 URL을 그대로 담고 서버에 별도 QR 레코드를 만들지 않는다. 인쇄물 사용을 전제로 정기 만료·갱신·재발급 UI를 제공하지 않으며, 해당 객실이나 접근 키가 삭제되지 않는 한 같은 QR을 계속 사용한다. 고객 웹 도메인이 바뀌면 기존 인쇄물도 새 주소를 알 수 없으므로 상업 인쇄 전에 운영 도메인을 확정해야 한다.

## Agent 페이지

- 공통 직원 로그인에서 Agent 역할 자동 이동
- ADMIN은 `/admin/agent`에서 전체 상담을 읽기 전용으로 조회하고 로고 오른쪽 `관리자 페이지` 버튼으로 Dashboard에 직접 이동
- LINE형 왼쪽 `Current chat room / Log` 목록과 오른쪽 채팅 본문
- 검색·호텔·언어 필터
- 새 문의 선택 시 별도 수락 화면 없는 대화 열기
- 한국어·일본어 Agent UI 전환
- 투숙객과 실시간 채팅
- 객실 정보, 상담 상태, 시작 시각과 남은 시간 표시
- 상담 종료
- 목록과 상세 헤더의 실제 담당 Agent 표시 및 미배정 문구
- 담당자 수락·종료 이벤트의 목록·상세 실시간 동기화

### Agent 화면 디자인 원칙

- 첨부된 LINE 데스크톱 UI처럼 왼쪽 목록과 오른쪽 본문을 전체 높이로 나눈다.
- 데스크톱에서는 목록과 본문 사이 구분선을 드래그해 목록 폭을 320~620px 안에서 조절하고, 조정값을 같은 브라우저에 저장한다. 키보드 방향키와 Home/End도 지원한다.
- 채팅 말풍선은 Agent 발신을 초록색 오른쪽, Guest 발신을 회색 왼쪽, 호텔 안내를 중앙 시스템 말풍선으로 표시한다.
- 연결 상태는 초록색, 남은 시간은 주황색, 종료 동작은 빨간색으로 의미를 일관되게 구분한다.
- 상담 목록은 대기·진행·종료 요약 카드와 상태별 표를 사용하고, 640px 이하에서는 표를 항목명이 포함된 카드 목록으로 전환한다.
- Agent 업무는 상담 목록과 채팅으로 이어지는 단일 흐름이므로 현재 `/agent`로 다시 이동하는 메뉴 하나뿐인 별도 탐색 사이드바를 표시하지 않는다.
- 화면 전체 너비의 상단 메뉴바 왼쪽에는 `REMOTE+` 로고만 두고, 오른쪽에는 `프로필 아이콘 → 직원 이름 → UI 언어 → 알림음 → 브라우저 알림 → 비밀번호 변경 → 로그아웃` 순서로 한 줄 배치한다. 오른쪽 메뉴 그룹은 자동 여백으로 우측 정렬하고 프로필과 이름은 DOM에서도 첫 사용자 정보 그룹으로 묶는다.
- ADMIN 전용 Agent 조회 화면에서는 로고 바로 오른쪽에 `관리자 페이지` 버튼을 추가한다. 일반 Agent 화면에서는 이 버튼을 만들지 않으며, ADMIN과 AGENT의 경로 접근 검사를 각각 `/admin`·`/admin/agent`와 `/agent`로 분리한다.
- 상담 목록 사이드바에서는 공통 메뉴를 완전히 제거하고 `Current chat room / Log`, 검색, 호텔·언어 필터, 상담방 목록 순서로 바로 시작한다.
- 상단 Agent 정보는 직원 이름을 보여 여러 직원이 공용 PC를 사용해도 현재 계정을 구분할 수 있게 한다. 좁은 화면에서는 로고 행과 오른쪽 메뉴 행을 분리하거나 버튼 문구를 아이콘으로 축약하되 오른쪽 그룹 안의 프로필 우선 순서는 유지한다.
- 640px 이하에서는 세 개의 요약 카드를 한 줄의 간결한 지표로 표시한다.
- 모바일 채팅은 동적 뷰포트 전체를 사용하고 안전 영역을 반영해 주소창·가상 키보드가 나타나도 입력창을 유지한다.
- Agent와 Guest 모두 메시지 목록만 내부 스크롤하며 입력·전송 영역은 화면 최하단에 고정한다.
- Agent 상담 본문은 조건부 읽기 전용 안내·오류와 무관하게 `헤더 / 메시지 본문 / 입력창` 세 행을 유지하며, 짧은 대화는 입력창 바로 위에 정렬한다.
- 상담 헤더 왼쪽에는 호텔·객실·상태·담당 Agent를 표시하고 오른쪽 action 그룹에는 `남은 시간 → 상담 종료` 순서로 두며 `margin-left: auto`로 끝 정렬한다. ADMIN 읽기 전용 화면에서는 종료 버튼을 표시하지 않는다.
- 채팅 또는 Log 기록을 선택하지 않은 빈 화면은 안내 문구만 중앙 정렬하고 초록색 R 로고 요소를 렌더링하지 않는다.
- 상담 종료 뒤에는 현재 본문을 읽기 전용으로 유지하고 Agent가 직접 `Log`를 선택하기 전에는 탭을 자동 전환하지 않는다.
- 대화방을 연 뒤 첫 Agent 답변 전에는 `첫 답변 후 15분`을 표시하고 입력을 허용한다. 첫 메시지가 저장되면 `chat:session-updated`의 서버 `expiresAt`으로 15분 카운트다운을 시작한다.
- 타이머 표시는 서버 정책 상한인 15분을 넘지 않게 제한해 PC·서버 시계 오차가 `17:00` 같은 값으로 노출되지 않게 한다.
- 새 메시지와 상담 종료 상태가 반영되면 채팅 영역 내부만 마지막 메시지로 이동하고 전체 페이지 스크롤은 변경하지 않는다.
- 새 대기 상담과 Agent에게 배정된 진행 상담의 고객 메시지는 데스크톱·모바일 모두 화면 위에 8초간 유지되는 팝업으로 호텔·객실과 메시지 미리보기를 보여준다.
- 브라우저 시스템 알림은 Agent가 `브라우저 알림 켜기`를 눌러 권한을 허용한 경우에만 사용하고, 현재 탭이 보이지 않을 때 화면 팝업을 보완한다.
- 비활성 Agent 탭은 새 상담·고객 메시지가 오면 제목을 번갈아 표시하고, 탭 또는 창에 다시 초점이 오면 원래 제목으로 즉시 복구한다.
- 진행 상담 목록은 마지막 메시지 시각 내림차순으로 정렬해 최근 대화가 메신저 목록처럼 맨 위에 보이게 한다.
- 언어 필터 옵션은 표시 문구와 별개로 실제 언어 코드를 명시하고, 상담의 언어 코드를 정규화해 `JA` 표시와 `ja` 데이터가 불일치하지 않게 한다.
- Guest가 `chat:session-updated`로 Agent 배정을 받아도 `상담원이 연결되었습니다. 개인정보나 결제 비밀번호는 전송하지 마세요.` 안내문을 채팅 화면에 추가하지 않는다. 내부 WebSocket 연결 상태와 입력 가능 상태 갱신은 그대로 유지한다.
- Guest 채팅 헤더에는 호텔명과 객실만 표시하고 우측 상단 연결 상태 문구·표시는 노출하지 않는다.
- 고객 모바일 채팅을 우선 UI로 보고, 460px 이하에서는 바깥 카드·그림자를 제거한 전체화면 메신저 형태로 호텔·객실 헤더, 말풍선 목록, 44px 이상 입력·전송 영역을 제공한다.
