# MVP 승인 체크리스트

- [x] 유효한 QR 링크로 투숙객 화면에 접속할 수 있다.
- [x] 유효하지 않은 접근 키를 서버가 거부한다.
- [x] 투숙객이 새 상담 세션을 만들 수 있다.
- [x] 상담원이 대기 상담을 확인할 수 있다.
- [x] 양쪽에서 실시간 텍스트 메시지를 송수신할 수 있다.
- [x] 새로고침 또는 일시 단절 후 진행 상담으로 재접속할 수 있다.
- [x] 서로 다른 세션의 메시지가 섞이지 않는다.
- [x] 15분 후 서버가 세션을 자동 종료한다.
- [x] 종료된 세션에서 추가 메시지를 보낼 수 없다.
- [x] 상담원이 상담을 직접 종료할 수 있다.
- [x] 인증되지 않은 사용자는 상담원 화면에 접근할 수 없다.
- [x] 서버가 과도한 메시지 요청을 제한한다.
- [x] 한 상담의 서버 오류가 다른 세션에 영향을 주지 않는다.
- [x] 주요 오류와 상담 상태 변경이 로그에 남는다.

검증 근거: 서버 단위 테스트, Phase 1·2·4/5 통합 테스트, 투숙객–Agent 브라우저 E2E, 관리자 브라우저 E2E.
